Why is this an issue?

should not use: attr("disabled", true)/attr("disabled", false)/attr("disabled", "disabled")/attr("disabled") attr("checked", true)/attr("checked", false)/attr("checked", "checked")/attr("checked")

use prop instead of attr

Noncompliant code example

    attr("disabled", true)/attr("disabled", false)/attr("disabled", "disabled")/attr("disabled")
    attr("checked", true)/attr("checked", false)/attr("checked", "checked")/attr("checked")

Compliant solution

    prop("disabled", true)/prop("disabled", false)/prop("disabled", "disabled")/prop("disabled")
    prop("checked", true)/prop("checked", false)/prop("checked", "checked")/prop("checked")
